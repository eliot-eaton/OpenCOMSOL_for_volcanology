# Contribution Templates

This directory contains templates to help you contribute to the OpenCOMSOL for Volcanology repository.

## Available Templates

### 1. MODEL_README_TEMPLATE.md
Use this template when submitting a COMSOL model (published or unfinished).

**How to use:**
1. Copy this template to your model's directory
2. Rename it to `README.md`
3. Fill in all applicable sections
4. Delete sections that don't apply (e.g., Citation for unpublished models)
5. Customize as needed for your specific model

### 2. IDEA_TEMPLATE.md
Use this template when proposing an idea for a future model.

**How to use:**
1. Copy this template to the `model_ideas/` directory
2. Rename it to match your idea (e.g., `lava_lake_convection.md`)
3. Fill in all relevant sections
4. Submit via pull request or issue

## Tips for Using Templates

- **Be thorough**: The more information you provide, the more useful your contribution
- **Be clear**: Write for an audience that may not be familiar with your specific research
- **Be honest**: Clearly state limitations, assumptions, and development status
- **Be collaborative**: Include contact information if you're open to collaboration

## Need Help?

- Review the main [CONTRIBUTING.md](../CONTRIBUTING.md) for detailed guidelines
- Look at existing models in the repository for examples
- Open an issue if you have questions

## Customization

These templates are starting points. Feel free to:
- Add sections relevant to your specific model
- Remove sections that don't apply
- Reorganize content to better suit your needs
- Include additional documentation formats (PDFs, images, etc.)

The goal is clear, comprehensive documentation that helps the community use and build upon your work.
